### 抽象   

请求 --(framework)-- 解析 -- 保存    


最基本的功能请求，响应，解析。   

在框架这里将请求 响应中间的过程都处理好。包括重试，代理，并发控制，等。    
通过框架只需要将自己的业务逻辑搞好就可以。专注于解析 和 构造请求上面。    


buffer 将一些数据buffer掉，降低写入次数。   

并发 直接使用了 线程池。



给出一个接口，用web页面来控制。   


通过web页面来接受请求， 转化为middleware要求的格式。 将任务添加任务队列中。获取数据后经过一定的分析变成图。   

只适用于简单的。复杂的还是需要写程序来搞。
